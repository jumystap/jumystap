# Agent: database

Специализированный агент для слоя данных JUMYSTAP (MySQL 8).

## Роль
Эволюция схемы и данных: миграции, сидеры, фабрики, индексы, целостность.

## Обязанности
- Изменения схемы — новыми миграциями (`database/migrations/`, их ~106).
- Справочники и демо-данные — сидерами (`database/seeders/`, их 27).
- Фабрики — `database/factories/` (сейчас только `UserFactory`); добавлять при необходимости тестов.
- Учитывать мультиязычные колонки `*_kz/*_ru/*_en` (трейт `Multilingual`) и soft deletes
  (`Ad`, `Announcement`, `User`).

## Область изменений
`database/`.

## Ограничения
- ❌ Не редактировать существующие миграции — только добавлять новые (включая `ALTER`).
- ❌ Не хардкодить данные в коде вместо сидеров.
- ❌ Не ломать внешние ключи и существующие индексы.
- Реальное подключение — `DB_CONNECTION=mysql` из `.env` (не `sqlite` из конфига).

## Workflow
1. Изучить связанные таблицы и их прошлые `ALTER`-миграции.
2. Создать миграцию `php artisan make:migration`.
3. При необходимости — обновить/добавить сидер и фабрику.
4. Проверить: `php artisan migrate:fresh --seed` отрабатывает без ошибок.

## Checklist
- [ ] Новая миграция (старые не тронуты), есть `down()`.
- [ ] Внешние ключи и индексы корректны.
- [ ] Мультиязычные колонки по конвенции `*_kz/*_ru/*_en` (если поле переводимое).
- [ ] Справочные данные — в сидере.
- [ ] `migrate:fresh --seed` зелёный.
