# Memory: Common mistakes (чего избегать)

Типовые ошибки при работе с этим проектом. Дополняй по мере обнаружения.

- ❌ **Писать логику/запросы в контроллере.** Логика → Service, запросы → Repository.
- ❌ **Вводить Resources/Policies/Events/DTO/Actions.** Их в проекте нет; используй принятые паттерны.
- ❌ **Добавлять строку в одну локаль.** Нужны и `public/locales/ru`, и `public/locales/kz`,
  плюс регистрация namespace в `ns` (`resources/js/i18n.js`).
- ❌ **Редактировать старые миграции.** Только новые миграции; старые трогать нельзя.
- ❌ **Полагаться на sqlite по умолчанию.** Рабочая СУБД — MySQL из `.env`.
- ❌ **Менять Blade-экран как React (и наоборот).** Сначала определить тип админ-экрана.
- ❌ **Магические строки статусов/ролей.** Только enums (`AdStatus`, `Roles`, ...).
- ❌ **Обращаться к `name_ru` напрямую**, когда есть локализованный `$model->name` (Multilingual).
- ❌ **Подмешивать новые UI-библиотеки.** Использовать уже принятые (antd/tailwind/headless/lucide).
- ❌ **Реальные вызовы внешних API в тестах** (Bitrix/SMSC/Gender) — мокать.
- ❌ **Коммитить секреты** (`SMSC_*`, `BITRIX_URI`, `GENDER_API_*`, `APP_KEY`).
- ❌ **Копировать опечатки в именах** (`Adress`, `Equimpments`) в новые сущности (существующие — не
  переименовывать).
