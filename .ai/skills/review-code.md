# Skill: review-code

Ревью изменений JUMYSTAP перед мержем.

## Когда использовать
Перед завершением задачи / подготовкой PR; по запросу «проверь дифф».

## Последовательность
1. Получи дифф (`git diff`, `git diff master...HEAD`).
2. Пройди по checklist (см. ниже), опираясь на `DEVELOPMENT_RULES.md` и `.ai/context/security-rules.md`.
3. Зафиксируй находки списком (файл:строка + проблема + предложение). Не правь без запроса.

## Checklist
- [ ] Слоистость: контроллер тонкий, логика в Service, запросы в Repository (нет SQL в контроллере).
- [ ] Валидация через FormRequest; кастомные правила в `app/Rules/`.
- [ ] Статусы/роли через enums; нет «магических» значений.
- [ ] Переводимые поля через Multilingual; UI-строки во всех локалях (ru/kz) + namespace.
- [ ] Схема — новой миграцией; FK/индексы целы; справочники в сидерах.
- [ ] Нет лишних слоёв (Resources/Policies/Events/DTO) и новых UI-библиотек.
- [ ] Тесты добавлены/обновлены в правильном каталоге; прогон зелёный.
- [ ] Безопасность: нет секретов, ввод валидируется, вывод санитизируется (dompurify).
- [ ] Несвязанная бизнес-логика не затронута; нет рефакторинга без запроса.
- [ ] Pint/Pest/`npm run build` проходят.

## Expected output
Структурированный отчёт ревью с конкретными ссылками на файлы/строки и рекомендациями;
явный вердикт: готово к мержу / требует правок.
