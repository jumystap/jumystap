# Skill: fix-bug

Исправление бага в JUMYSTAP.

## Когда использовать
Воспроизводимый дефект в поведении (бэкенд или фронтенд).

## Последовательность
1. **Воспроизведи:** шаги, входные данные, ожидаемое vs фактическое. Определи слой (Controller/
   Service/Repository/Model/Frontend/Blade).
2. **Корневая причина:** не маскируй симптом. Пройди цепочку Controller → Service → Repository →
   Model или Page → props. Для фронта проверь, Blade это или React-страница.
3. **Тест на регресс:** напиши падающий тест, воспроизводящий баг (Feature/Unit по типу).
4. **Минимальная правка:** исправь в правильном слое, без не связанных изменений и рефакторинга.
5. **Зелёный тест:** убедись, что новый тест проходит и старые не сломались.

## Обязательные проверки
- `./vendor/bin/pest` (новый тест зелёный, регрессий нет).
- `./vendor/bin/pint`
- `npm run build` + проверка в браузере (если фронтенд), на ru и kz.

## Expected output
Баг устранён в корне, покрыт регресс-тестом, дифф минимален. Если причина неочевидна или
системна — добавь запись в `.ai/memory/common-mistakes.md`.
