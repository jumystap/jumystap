# Skill: refactor-module

Безопасный рефакторинг модуля JUMYSTAP.

> ⚠️ Рефакторинг выполняется ТОЛЬКО по явному запросу пользователя. Без запроса — не рефактори
> «заодно». Бизнес-поведение не меняется.

## Когда использовать
Пользователь явно просит улучшить структуру модуля без изменения поведения (например, вынести
логику из контроллера в Service, разнести запросы в Repository).

## Последовательность
1. **Зафиксируй поведение тестами:** убедись, что есть тесты на текущее поведение; если нет —
   сначала напиши характеризующие тесты (см. `write-tests.md`).
2. **Картируй цепочку:** Controller → Service → Repository → Model — что куда относится.
3. **Маленькие шаги:** переноси по одному ответственному блоку; после каждого — прогон тестов.
4. **Сохрани контракты:** сигнатуры публичных методов, маршруты, имена Inertia-страниц/props.
5. **Не меняй схему БД и бизнес-правила** в рамках рефакторинга.

## Обязательные проверки
- `./vendor/bin/pest` после каждого шага (поведение неизменно).
- `./vendor/bin/pint`
- `npm run build` (если затронут фронтенд).

## Expected output
Та же функциональность, чище слои; тесты зелёные до и после; дифф не содержит изменений поведения.
Запиши значимые решения в `.ai/memory/architectural-decisions.md`.
