# AGENTS — JUMYSTAP

Роли AI-агентов для долгосрочной разработки (Claude Code, Codex, прочие агенты). Этот файл —
точка входа для агентских инструментов; детальные инструкции каждой роли — в `.ai/agents/`.

**Общие правила для всех агентов:**
- Соблюдай `DEVELOPMENT_RULES.md` и обязательный workflow из `CLAUDE.md`.
- Для задач через Codex используй `.ai/agents/codex.md` как coding guide.
- Не меняй несвязанную бизнес-логику, не рефактори без явного запроса, не удаляй файлы.
- Перед изменением — изучи существующую реализацию и найди похожий пример.
- После изменения — `./vendor/bin/pint`, `./vendor/bin/pest`, `npm run build` (что применимо).
- Фиксируй решения и подводные камни в `.ai/memory/*`.

## codex-agent
**Детали:** `.ai/agents/codex.md`

- **Ответственность:** общий fullstack workflow Codex: анализ, план, минимальная реализация,
  проверки и краткая отчётность по изменениям.
- **Область изменений:** зависит от задачи; перед правкой Codex обязан назвать конкретные файлы.
- **Ограничения:** не трогать `.env`, секреты, production configs и несвязанную бизнес-логику;
  при риске сломать API/UI — остановиться и объяснить риск.
- **Критерии качества:** изменения соответствуют `DEVELOPMENT_RULES.md`; применимые проверки
  запущены или причина пропуска явно названа.

## backend-laravel-agent
**Детали:** `.ai/agents/backend-laravel.md`

- **Ответственность:** контроллеры, сервисы, репозитории, модели, enums, FormRequest, миграции,
  команды, jobs, Telegram-handler, интеграции (Bitrix/SMSC/Gender).
- **Область изменений:** `app/`, `routes/`, `database/migrations/`, `config/` (осторожно).
- **Ограничения:** тонкие контроллеры; логика в Service; запросы в Repository; валидация в
  FormRequest; статусы через Enum; без Resources/Policies/Events/DTO. Не менять схему правкой
  старых миграций.
- **Критерии качества:** Pint чист, Pest зелёный, новый код покрыт тестом, нет SQL в контроллерах.

## frontend-agent
**Детали:** `.ai/agents/frontend-react.md`

- **Ответственность:** React-страницы (Inertia), компоненты, layouts, i18n, Blade-админка.
- **Область изменений:** `resources/js/`, `resources/views/`, `public/locales/`, `tailwind.config.js`.
- **Ограничения:** state через Inertia-props/useState; формы через react-hook-form; не вводить новые
  UI-библиотеки; строки — во все локали (ru/kz) и в зарегистрированный namespace.
- **Критерии качества:** `npm run build` проходит; UI проверен в браузере на ru и kz; нет регрессий.

## database-agent
**Детали:** `.ai/agents/database.md`

- **Ответственность:** миграции, сидеры, фабрики, индексы, целостность данных.
- **Область изменений:** `database/`.
- **Ограничения:** только новые миграции (старые не править); справочники — сидерами; учитывать
  мультиязычные колонки `*_kz/*_ru/*_en` и soft deletes.
- **Критерии качества:** `migrate:fresh --seed` отрабатывает; внешние ключи и индексы корректны.

## qa-agent
**Детали:** `.ai/agents/qa-review.md`

- **Ответственность:** тесты Pest (Feature/Unit), проверка сценариев, регрессий.
- **Область изменений:** `tests/`.
- **Ограничения:** Feature — HTTP/Inertia + RefreshDatabase; Unit — Repositories/Services/Controllers;
  использовать тестовое окружение из `phpunit.xml`.
- **Критерии качества:** новые ветки логики покрыты; тесты детерминированы; зелёный прогон.

## devops-agent
**Детали:** `.ai/agents/devops.md`

- **Ответственность:** Docker, nginx, entrypoint, очереди/планировщик, окружение, (будущий) CI/CD.
- **Область изменений:** `Dockerfile`, `docker-compose.yml`, `docker/`, `.env.example`, `.github/` (TODO).
- **Ограничения:** не коммитить секреты; не ломать совместимость с `database`-драйверами очереди/кэша/сессий.
- **Критерии качества:** `docker compose up` поднимает app/nginx/queue/scheduler/mysql; health `/up` ок.

## code-review-agent
**Детали:** `.ai/agents/code-review.md`

- **Ответственность:** ревью диффа на соответствие правилам, безопасность, корректность.
- **Область изменений:** только комментарии/замечания, без правок без запроса.
- **Ограничения:** проверять по `DEVELOPMENT_RULES.md` и `.ai/context/security-rules.md`.
- **Критерии качества:** найдены нарушения слоёв, утечки секретов, отсутствие тестов, проблемы локализации.
